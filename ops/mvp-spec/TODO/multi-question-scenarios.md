# Multi-Question Scenarios Implementation Plan

## Overview

Scenarios currently have no questions field. The prompt text is dynamically generated at runtime by concatenating context fields. This plan adds a `questions` array to scenarios, generated by the AI worker with increasing difficulty, and modifies the practice flow to step through them one at a time.

```python
def _build_scenario_prompt_text(context: ScenarioContextView) -> str:
    sections = [
        "Work through the following workplace scenario in a single written response.",
        f"Business context: {context.business_context}",
        f"Learner objective: {context.learner_objective}",
    ]
    if context.constraints:
        sections.append(f"Constraints: {_join_items(context.constraints)}")
    if context.stakeholder_tensions:
        sections.append(f"Stakeholder tensions: {_join_items(context.stakeholder_tensions)}")
    if context.mock_company is not None:
        sections.append(
            "Company context: "
            f"{context.mock_company.name} ({context.mock_company.industry}) - "
            f"{context.mock_company.operating_context}"
        )
    if context.mock_people:
        sections.append(
            "Stakeholders: "
            + " | ".join(
                f"{person.name} ({person.role}): {person.relationship_to_scenario}"
                for person in context.mock_people
            )
        )
    sections.append("Respond with the action or message you would deliver next.")
    return "\n".join(sections)
```

## Implementation Order

1. **Database Migration** - Add `questions` column to `scenarios` table
2. **Domain Models** - Update Commands/Views/Drafts to include `questions`
3. **Generation** - Modify LLM prompts to output questions with increasing difficulty
4. **Validation** - Update worker validation to check question count
5. **Persistence** - Store questions in ScenarioRecord
6. **Runtime Query** - Use stored questions in prompt building
7. **Submit Flow** - Handle step progression in submission
8. **Frontend** - Display questions and handle progression

---

## Backward Compatibility

- If `questions` is empty, fall back to current behavior (single open-ended prompt)
- Existing scenarios continue to work without modification
- New scenarios generated after this change will have questions
