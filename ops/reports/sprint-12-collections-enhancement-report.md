## What Worked Well

- Upsert pattern for ratings worked cleanly - existing rating record updated, new record inserted if none exists
- Transactional denormalization ensures avg_rating consistency - rating operation and update happen in same transaction
- Stageflow pipelines reused from existing collection service patterns, providing consistency
- Alembic autogenerate produced clean migration after fixing SQLite FK constraint issue
- Discovery tier logic in `discovery_tier_for_collection()` required no changes; filters handle tier correctly

## Challenges And Friction

- SQLite FK constraint limitation: autogenerated migration tried to add FK constraint via ALTER TABLE which SQLite does not support
  - Root cause: Alembic autogenerate detected existing FK but SQLite requires batch mode for constraint changes
  - Impact: Had to manually edit migration to remove FK alter and constraint statements
  - Fix: Simplified migration to only add columns and table, no constraint changes
- Import ordering issues: Multiple edits caused ruff I001 import block sorting warnings
  - Root cause: Adding imports in wrong order during edits
  - Impact: Required running `ruff check --fix` to resolve
  - Fix: Ensured proper import grouping in all affected files
